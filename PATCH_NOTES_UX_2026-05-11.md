`admin-styles.css` 에 `.card / .card-header / .card-body` 클래스 본체가
아예 정의돼 있지 않았습니다. HTML 은 카드 구조로 작성됐지만 실제 스타일이
없어 모든 섹션이 경계 없는 평평한 박스로 흘러내리고 있었어요.
인라인 스타일(border, border-radius 등)이 일부 임시 처방으로 들어가 있었지만
일관성 없고 다크모드 대응이 불가능한 상태였습니다.

## 변경 내용

### 1) 카드 시스템 정식 정의 (admin-styles.css)
- `.card / .card-header / .card-body` 본체 정의 — 다크모드 자동 대응(--surface, --border, --text 토큰 사용)
- `.card[data-accent="info|warn|brand|success"]` — 좌측 3px 컬러 액센트
  - **파랑(info)**: 지역 선택 (정보 입력)
  - **노랑(warn)**: API 사용량 (한도 모니터링)
  - **주황(brand)**: 수집 실행 (액션)
  - **초록(success)**: 수집 현황 (결과)

### 2) 진행 상태 칩 — C안 (미니바 + 분수)
구 `chip()` 함수가 인라인 hex 색을 박아넣던 방식 → 새 `pchip()` 함수가
CSS 클래스(`.bld-pchip--idle/low/mid/done/has`)만 결정. 다크모드 자동 대응.

칩 구조:
```
┌──────────────────────┐
│ 전유+공용  114/1,158 │  ← 분수 표시 (분모로 무엇이 기준인지 명확)
│ ███───────────────── │  ← 미니바 (시각적 진행도)
└──────────────────────┘
```

색상 의미:
- **녹색**: 95%↑ (사실상 완료)
- **노랑**: 50%↑ (진행 중)
- **빨강**: 0~50% (미흡)
- **회색**: 0건 (미수집)
- 호버 시 `tooltip` 으로 "표제부: 4,957 / 4,957 (100%)" 형식 노출

### 3) 사용량 카드 — 주요/보조 2그룹 분리
11개 엔드포인트가 똑같은 무게로 나열되던 화면을:
- **주요 엔드포인트** (5개): 표제부 · 전유+공용 · 지오코딩 · 총괄표제 · 부속지번
- **보조 엔드포인트** (6개): 층별·지역·오수·공시가·기본·전유

각 그룹은 좌측 3px 세로 막대로 시각적 구분. 색상 인라인 hex 모두 제거 →
`data-state="idle|low|mid|high"` 속성만으로 CSS 클래스가 색 결정 → 다크모드 자동.

### 4) 최근 7일 그래프
- 상단에 **오늘 / 일평균 / 최고** 3개 통계 한 줄 추가
- 막대 위 호버 시 숫자 노출 (커서 올리면 보이는 툴팁)
- **오늘 막대**는 브랜드 컬러 + 외곽 글로우로 강조
- 0건 막대는 옅은 회색으로 차분히 표시 (이전엔 보이지 않음 → 날짜 라벨만 나옴)

### 5) 지역 선택 / 일괄 실행 / 로그 영역 정돈
- 지역 선택: 시도/시군구/버튼/카운트를 한 줄 toolbar 로 묶고 미세 배경
- **선택 카운트** 칩: 0개일 때 회색, 1개+ 일 때 브랜드 컬러로 강조
- 일괄 실행 체크박스: 체크된 항목은 라벨이 브랜드 컬러+굵게로 강조
- 로그 영역: 모노스페이스 폰트 + 미세 배경으로 콘솔 느낌

### 6) 모바일 반응형 (768px / 480px 브레이크포인트)
- 768px↓: 카드 헤더/바디 패딩 축소, 사용량 카드 가로폭 축소, 엔드포인트명 숨김
- 480px↓: 사용량 그리드 2열 고정, 진행 칩 2열 고정, 시도/시군구 드롭다운 풀너비
